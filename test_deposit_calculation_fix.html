Test: Deposit Calculation Fix

❌ ISSUE IDENTIFIED

Problem: When adding check-in and duration to Reserved bookings, it automatically shows "Payment - ₱1000.00" even though no payment has been added yet.

Expected: Should show "Payment - ₱0.00" until actual payment is made.

Root Cause: Code was using booking.deposit for Reserved bookings, but Reserved bookings store payment info in downpayment_cash, downpayment_gcash, downpayment_maya fields.

✅ FIX APPLIED

Solution: Updated deposit calculation logic to handle Reserved bookings differently:

Before (WRONG):

// Calculate deposit
const deposit = parseFloat(booking.deposit) || 0;

After (FIXED):

// Calculate deposit - handle Reserved bookings differently
let deposit = 0;
if (booking.status === 'Reserved') {
    // For Reserved bookings, check if there's actual payment made
    const downpaymentCash = parseFloat(booking.downpayment_cash) || 0;
    const downpaymentGCash = parseFloat(booking.downpayment_gcash) || 0;
    const downpaymentMaya = parseFloat(booking.downpayment_maya) || 0;
    deposit = downpaymentCash + downpaymentGCash + downpaymentMaya;
} else {
    // For non-Reserved bookings, use regular deposit
    deposit = parseFloat(booking.deposit) || 0;
}
