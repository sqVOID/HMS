Test: Reservation Mode Deposit Fix ✅

❌ ISSUES IDENTIFIED

Issue 1: Auto-display of ₱1000 Payment

Problem: When selecting duration in Reservation mode, it automatically shows "Payment - ₱1000.00" even though no payment has been made yet.

Expected: Should show "Payment - ₱0.00" until actual payment is entered.

Issue 2: Incorrect Subtotal Calculation

Problem: When editing payment to ₱500.00, subtotal still shows ₱1000.00 instead of ₱1500.00.

Expected: Subtotal should be ₱2000.00 - ₱500.00 = ₱1500.00

Root Cause:

The code was checking for Reserved status but not for Reservation mode. When creating a new booking in Reservation mode, the booking object might have old data from a previous booking on that room.

✅ FIX APPLIED

Solution:

Added check for Reservation mode in addition to checking booking status:

// Check if we're in Reservation mode (creating new booking)
const reservationRadio = document.getElementById('editReservationRadio');
const isReservationMode = reservationRadio && reservationRadio.checked;

// For Available bookings OR Reservation mode without confirmed payment
if (isAvailable || (isReservationMode && !isReserved)) {
    deposit = 0;  // ✅ Set to 0 for new bookings
}

Key Logic:

  • isAvailable: Booking status is "Available" or empty
  • isReservationMode: User selected "Reservation" radio button
  • !isReserved: Booking status is NOT "Reserved" yet
  • Result: If in Reservation mode but not yet Reserved, deposit = 0
